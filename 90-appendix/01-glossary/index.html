    용어집 (Glossary)

    TL;DR

    • 본 문서에서는 프로젝트 구성원 간의 원활한 소통을 위해 사용되는 핵심 용어를 정의합니다.

    📗 핵심 용어 정의

    용어 (KR) 용어 (EN) 설명
    작업 WorkOrder (WO) 설치 또는 AS를 수행하는 최소 관리 단위.
    권한/역할 RBAC / Role Admin, Team Manager, Technician 등 사용자 등급.
    현장 Site 제품이 실제로 설치되어 있는 장소 (고객사의 지점 등).
    고객사 Customer 계약의 주체이며 여러 현장(Site)을 소유할 수 있는 조직.
    점검 항목 Checklist Item 현장에서 기사가 확인해야 할 개별 질문 (PASS/FAIL/NA).
    증빙 데이터 Evidence 현장 서명, 사진 등 작업 완료의 근거가 되는 데이터.
    상태 전이 State Transition 작업의 진행 단계(DRAFT ➔ COMPLETED 등) 변화.
    감사 로그 Audit Log 시스템 내에서 발생한 모든 주요 행위의 기록.

    🎨 유관 개념 설명

    • 배정 체인 (Assignment Chain): 본사(팀 배정) ➔ 팀 관리자(기사 배정)로 이어지는 관리 흐름입니다.
    • 자동 저장 (Autosave): 현장 기사가 입력 중 데이터 소실을 방지하기 위해 실시간으로 서버에 저장되는 기능입니다.
    • 문서화 (Documentation): 작업 완료 후 데이터가 결합되어 생성되는 최종 보고서 PDF를 의미합니다.
