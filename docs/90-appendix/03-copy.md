# 용어/문구 세트

## TL;DR
- 화면/상태/버튼 문구를 통일해 사용자 혼란과 구현 오류를 줄입니다.

## 본문
기사 화면(M-02) 현장형 문구 세트 v0.2 (초단문)
1) 자동 저장(autosave)
• 저장 중(표시는 최소):
o 저장중… (옵션: 표시 안 해도 됨)
• 저장 완료(옵션):
o 저장됨
• 저장 실패(필수):
o 저장 실패 · 재시도
재시도 버튼/링크:
• 재시도
네트워크 안내(선택):
• 네트워크를 확인해주세요

2) 필수 누락/검증 메시지(완료 제출 시)
• 필수 항목 미완료:
o 필수 항목이 남아있어요
• 이상(FAIL)인데 메모 없음:
o 이상 항목 메모가 필요해요
• 서명 미완료:
o 서명을 먼저 받아주세요
• 서명자 이름 없음:
o 서명자 이름을 입력해주세요
실패 시 자동 스크롤: 해당 섹션으로 이동 + 빨간 테두리 강조(UX 규칙).

3) 서명 관련
• 서명 안내(서명 패드 위 작게):
o 고객에게 화면을 건네 서명받아 주세요
• 서명 저장 버튼:
o 서명 저장
• 저장 성공 토스트:
o 서명 저장 완료
• 저장 실패:
o 서명 저장 실패 · 재시도
• 서명 다시 받기:
o 서명 다시 받기
• 서명 지우기:
o 서명 지우기
서명 패드 빈 상태 경고(선택):
• 서명이 비어있어요

4) 사진 업로드(권장)
• 섹션 설명(짧게):
o 전/후 사진을 남겨주세요(권장)
• 업로드 중:
o 업로드중…
• 업로드 완료:
o 업로드 완료
• 업로드 실패:
o 업로드 실패 · 재시도
• 삭제 확인 모달:
o 제목: 사진 삭제할까요?
o 버튼: 삭제 / 취소
(나중에 사진 필수 토글 켤 때)
• 전/후 사진이 필요해요

5) 완료 제출(핵심)
완료 버튼:
• 완료 제출
완료 확인 모달(선택, 초반엔 OFF 권장)
• 제목: 완료 처리할까요?
• 설명: 제출 후 수정이 제한될 수 있어요
• 버튼: 취소 / 제출
완료 성공:
• 완료 처리됐어요
완료 실패:
• 완료 실패 · 다시 시도해주세요
중복 제출 방지(연타 시):
• 처리중…

6) 고객 발송(선택)
섹션 설명:
• 완료 문서를 고객에게 보낼 수 있어요
형식 오류:
• 전화번호: 번호를 확인해주세요
• 이메일: 이메일을 확인해주세요
발송 체크 안내(선택):
• 완료 후 자동 발송돼요

7) 접근/세션 오류(필수)
• 권한 없음(다른 사람 작업 접근):
o 이 작업은 볼 수 없어요
• 로그인 만료:
o 로그인이 만료됐어요 · 다시 로그인
• 서버 오류(일반):
o 잠시 후 다시 시도해주세요

8) 상태 배지(기사용)
• 배정됨
• 진행중
• 완료

용어/톤 통일 규칙(Phase 1)
• PASS/FAIL은 화면에 노출하지 않음 → OK/이상/해당없음
• “오류” 대신 “실패” 사용(현장 스트레스↓)
• 문장형 대신 짧은 구(3~12자) 중심
