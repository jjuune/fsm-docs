# 범위(In/Out)

## TL;DR
- Phase 1에서 **반드시 구현할 것**과 **보류할 것**을 명확히 분리합니다.

## 본문
Phase 1 범위 정의 (In Scope / Out of Scope) v0.1
1) In Scope (Phase 1에서 반드시 구현)
1. WorkOrder(작업) 생성/조회: 고객/현장 연결, 요약/특이사항, 방문 일정 입력
2. 배정 체인 운영
o 본사(Admin): 팀 배정/변경(정책)
o 팀관리자(TM): 기사 배정/재배정(정책)
3. 기사 작업 수행(M-02)
o 체크리스트 입력(autosave) + FAIL 메모 필수
o 고객 서명 저장(서명자 이름 포함) 필수
o (권장) 사진 업로드(전/후)
o 완료 제출(서버 검증 포함)
4. 팀 운영 화면(T-01/T-02/T-03)
o KPI/현황 요약
o 미배정 중심 작업함 + 퀵 기사 배정
o 작업 상세에서 증빙 상태 조회
5. 본사 운영 화면(A-02/A-04)
o 팀 미배정 중심 목록 + 퀵 팀 배정
o 작업 상세에서 팀 변경/취소/문서/발송 상태 확인
6. PDF 보관(최소)
o 완료 후 PDF 생성/저장 또는 다운로드 가능한 형태로 보관(구현 방식은 선택)
7. 권한/스코프 적용
o ORG/TEAM/SELF 스코프 + 역할 기반 접근 제어

2) Out of Scope (Phase 1에서는 제외/보류)
1. 제품 재고/입출고/시리얼 단위 추적(ProductUnit, 재고 테이블 등) — Phase 2
2. 가격정책/프로모션/매출 통계 등 영업 관리 전체 — Phase 2/3
3. 정산/세금계산서/결제 연동 — Phase 2+
4. 멀티 테넌트(외부 고객사별 독립 운영) 구조 — Phase 3
5. 고급 권한 시스템(세부 Permission 테이블) — Phase 2 (Phase 1은 role로 충분)
6. 발송(문자/이메일) 고도화
o 실패 원인 분석/대량 발송/템플릿 관리 — Phase 2
(Phase 1은 “자동 발송/재발송 최소”만)
7. AuditLog 고도화(감사 리포트/내보내기) — Phase 2
(Phase 1은 핵심 이벤트 최소 기록 또는 간단 로그로 대체 가능)
8. 오프라인 모드(무신호 저장 후 동기화) — Phase 2
(Phase 1은 네트워크 불안 시 “재시도 UX” 중심)
9. SLA/알림/에스컬레이션(슬랙/푸시) — Phase 2

3) Phase 1 성공 기준(필드테스트 관점)
• 실제 기사 1~2명이 모바일 웹으로 체크리스트+서명 후 완료 제출을 문제없이 수행한다.
• 팀관리자가 미배정 → 기사 배정을 목록에서 빠르게 처리한다.
• 본사가 팀 미배정 작업을 목록에서 팀 배정하고 진행 상태를 확인한다.
• 완료 건에 대해 PDF(또는 동등한 문서 결과물)가 남아 추후 조회 가능하다.
