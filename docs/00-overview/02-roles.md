# 역할과 책임(RBAC)

## TL;DR
- 3개 역할(Admin/TeamManager/Technician) 기준으로 화면과 권한을 분리하고, WorkOrder 상태 전이를 권한으로 제어.

## 본문
### 역할 정의(초안)
- **Admin(본사 관리자)**: 작업 생성/팀 배정/작업 모니터링/내역 조회
- **Team Manager(팀 관리자)**: 작업 수락/기사 배정/팀 단위 진행 관리
- **Technician(기사)**: 현장 작업 수행, 체크리스트/사진/서명 수집, 완료 제출

> 상세 화면 기능은 `20-product/*` 문서에서 역할별로 정리.
