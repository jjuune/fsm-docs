# 프로젝트 정의

## TL;DR
- Phase 1은 설치·AS 내부 운영을 디지털화해 처리 속도/추적성/증빙 품질을 개선하는 것이 목표.

## 본문
프로젝트 정의 (1차 단계 – 내부 효율 개선 중심)
프로젝트 명 (초안)
공간살균기 설치·AS 내부 운영 디지털화 프로젝트 (Phase 1)
1️⃣ 프로젝트 목적 (Phase 1 한정)
본 프로젝트는 당사 설치 및 AS 업무의 수작업·분산 관리 체계를 디지털화하여,
현장 증빙(서명·사진·문서)을 표준화하고,
본사–팀–기사 간 작업 흐름을 통합 관리하는 내부 운영 시스템을 구축하는 것을 목적으로 한다.

2️⃣ 해결하려는 문제 정의 (Problem Statement)
현재 운영상의 문제:
• 설치·AS 증빙 문서의 누락/분산
• 현장 작업 이력 조회의 어려움
• 팀 단위 배정 체계의 비표준화
• 작업 완료 후 문서 전달의 수동 처리
• 관리자 실시간 모니터링 한계

3️⃣ 1차 목표 (Phase 1 Objectives)
운영 효율
• 작업 생성 → 배정 → 완료 흐름의 표준화
• 팀 단위 관리 체계 정립
증빙 체계
• 현장 서명 필수화
• 설치/AS 완료 문서 자동 생성
• 고객 대상 자동 발송 체계 구축
통제/조회
• 고객·현장·시리얼 기준 이력 조회 가능
• 관리자 대시보드로 상태 가시화

📌 프로젝트 범위 (Phase 1 Scope)
포함
• 사용자 권한 체계 (Admin / TeamManager / Technician)
• WorkOrder 기반 설치·AS 관리
• 체크리스트 및 서명 수집
• PDF 문서 자동 생성
• 이메일/문자 발송
• 작업 이력 조회 기능
제외 (Phase 2 이후)
• 영업 관리 모듈
• 가격 정책/프로모션
• 정산/회계 연동
• 고급 재고 시스템
• 외부 고객용 SaaS 기능

📌 성공 기준 (Phase 1 KPI)
• 문서 누락률 0%
• 설치/AS 이력 조회 시간 50% 이상 단축
• 작업 상태 실시간 파악 가능
• 고객 문서 발송 자동화 100%

📌 Phase 2, 3 전략 (예정)
Phase 2 (내부 고도화)
• 재고/시리얼 추적 고도화
• 영업 모듈 통합
• 통계/성과 분석
Phase 3 (외부 확장)
• 파트너사 통합 운영
• 멀티 조직 구조 확장
• SaaS 모델 전환 가능성 검토

🔥 전략적으로 중요한 점

업무 프로세스 표준화 및 디지털 전환 프로젝트
