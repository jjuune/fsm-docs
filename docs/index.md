# 문서 홈

## TL;DR
- Phase 1의 목표는 **수기/분산 관리**를 줄이고, **현장 증빙(서명·사진·문서)을 표준화**하며, **본사–팀–기사 흐름을 하나로 통합**하는 것입니다.
- 개발 착수는 **FSM → M-02 → 배정/조회 → 권한 → 데이터** 순이 가장 안전합니다.

## 문서 구성
- 00 Overview: 배경/목표/범위/리스크
- 10 Domain: FSM, 정책(체크리스트/검증/완료처리)
- 20 Product: 역할별 화면 스펙(관리자/팀/기사)
- 30 Implementation: M-02 통합 스펙, 컴포넌트/리스크, API, CRUD
- 40 Data: ERD, 필드 사전
- 90 Appendix: 용어/문구 세트, 변경이력
