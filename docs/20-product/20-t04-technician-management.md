# T-04 기사 관리 + T-00 팀 대시보드

## TL;DR
- **T-04:** TM이 기사를 초대·수정·비활성화하는 관리 화면. 삭제 금지, 팀 이동은 Phase 2.
- **T-00:** TM 홈 대시보드. 기사별 운영지표 MVP 카드. 성과평가 아님.

---

## 👷 1. T-04 기사 관리

### 1.1 기사 목록 화면
| 요소 | 내용 |
| :--- | :--- |
| 목록 표시 | 이름, 연락처, 상태(ACTIVE/INACTIVE), 현재 배정 건수 |
| 필터 | 상태 필터 (전체 / ACTIVE / INACTIVE) |
| 액션 | [기사 초대] 버튼, 각 행 클릭 → 기사 상세 |

### 1.2 기사 초대(생성) 폼
| 필드 | 필수 | 비고 |
| :--- | :---: | :--- |
| 이름(표시명) | ✅ 필수 | |
| 전화번호 | ✅ 필수 | |
| 로그인 이메일 | ✅ 필수 | 계정 생성용 |
| 임시 비밀번호 | ✅ 필수 | 최초 로그인 후 변경 권장 |
| 역할 | — | **role=TECH 고정** (선택 불가) |
| 팀 | — | **현재 TM의 팀 자동 배정** (변경 불가) |

> **Phase 1 제약:** 팀 이동(다른 팀으로 변경)은 Admin만 가능하며, Phase 2에서 TM에게 권한 부여 검토.

### 1.3 기사 상세/수정 화면
| 필드 | 수정 가능 여부 |
| :--- | :---: |
| 이름(표시명) | ✅ 가능 |
| 전화번호(연락처) | ✅ 가능 |
| 이메일 | ⚠️ (정책: Phase 1 제한) |
| 역할 | ❌ 불가 (TECH 고정) |
| 팀 | ❌ 불가 (Phase 2) |

### 1.4 기사 비활성화
| 항목 | 내용 |
| :--- | :--- |
| 액션 | "기사 비활성화" 버튼 (상세 화면 하단) |
| 사유 | 입력 창 제공 (선택 사항) |
| 확인 모달 | "비활성화된 기사는 로그인 및 신규 배정이 차단됩니다." |
| 삭제 금지 | **삭제 버튼 없음 (Phase 1)** |

## 📊 2. T-00 팀 대시보드 (운영지표 MVP)

### 2.1 카드 구성
| 카드 | 지표 | 정의 |
| :--- | :--- | :--- |
| 기사별 상태 카운트 | 배정됨 / 진행중 / 완료 / 지연 | 해당 기사에게 배정된 WO 상태별 건수 |
| 지연 카운트 | `scheduled_at < now()` AND `status != COMPLETED` | 예정일 지난 미완료 건수 |

### 2.2 인터랙션
| 동작 | 결과 |
| :--- | :--- |
| 기사 카드 클릭 | T-02(팀 작업함)으로 이동, `assigned_technician_id` 필터 자동 적용 |
| 지연 카운트 클릭 | T-02로 이동, "지연" 필터 자동 적용 |

> **통계는 "운영지표(MVP)"** 의미입니다. 개인 성과평가 용도로 사용하지 않습니다.

### 2.3 화면 구성
| 요소 | 내용 |
| :--- | :--- |
| 헤더 | "팀 현황 (오늘 기준)" + 마지막 갱신 시간 |
| 기사 카드 그리드 | 기사명, 상태별 카운트, 지연 카운트 |
| 상단 [기사 관리] 링크 | T-04로 이동 |
| [작업함] 링크 | T-02로 이동 |
