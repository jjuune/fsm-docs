# Team Manager(팀 관리자) 화면 스펙

## TL;DR
- **목적:** 팀 단위 실시간 모니터링 및 소속 기사 배정/재배정 관리
- **핵심 화면:** T-01(대시보드), T-02(작업함), T-03(작업 상세)

---

## 📊 T-01 팀 대시보드 (현황 파악)

### 1. 화면 개요
로그인 시 첫 화면으로, 팀의 오늘 목표와 병목 지점(미배정, 지연)을 즉시 파악합니다.

### 2. 주요 구성 요소
| 섹션 | 상세 설명 | 비고 |
| :--- | :--- | :--- |
| **KPI 요약** | 미배정, 오늘 예정, 진행중, 지연, 완료 건수 | 클릭 시 T-02 탭으로 연동 |
| **오늘 일정** | 시간순 작업 리스트 (최대 8건) | 미배정/지연 항목 최상단 노출 |
| **기사 현황** | 기사별 할당량, 진행중, 완료(오늘) 통계 | 기사별 부하 조절 용도 |
| **유형 분포** | 설치 vs AS 작업 비율 (단순 숫자) | - |

---

## 📂 T-02 팀 작업함 (리스트 및 퀵 배정)

### 1. 화면 개요
상태별로 작업을 분류하고, 특히 '기사 미배정' 작업을 가장 빠르고 직관적으로 배정하는 화면입니다.

### 2. 프리셋 탭 구성
- **[미배정]:** `status=TEAM_ASSIGNED` (기사 배정 필요) ➔ **기본 진입 탭**
- **[오늘]:** 오늘 날짜 일정 작업 전체
- **[진행중]:** 현재 현장 수행 중인 작업
- **[전체]:** 팀에 할당된 모든 작업 이력

### 3. 퀵 기사 배정
- 카드 내부의 **[기사 배정]** 버튼 클릭 ➔ 기사 선택 바텀시트 호출 ➔ 배정 완료 ➔ 리스트에서 즉시 사라짐 (미배정 탭 기준)

---

## 🕵️ T-03 작업 상세 (배정 및 증빙 확인)

### 1. 화면 개요
특정 작업의 기사 배정을 확정하고, 기사가 올리는 증빙 데이터(체크리스트, 사진)를 실시간 모니터링합니다.

### 2. 섹션별 상세 스펙
| 섹션 | 주요 내용 | 비고 |
| :--- | :--- | :--- |
| **요약 카드** | 현장 정보, 주소, 담당자 연락처, 일정 | 전화/지도 연동 |
| **기사 배정** | 담당 기사 선택 드롭다운 및 프로필 확인 | 배정 및 재배정(사유 필수) |
| **증빙 상태판** | 체크리스트, 서명, 사진, PDF, 발송 상태 | ✅/⚠️ 아이콘으로 상태 표시 |
| **데이터 상세** | 체크리스트 응답(이상 항목 하이라이트), 사진 갤러리 | 읽기 전용 (수정 제한) |

## 🛠️ UI 텍스트 목업 (Mockup Text)

### [T-01 대시보드]
- **KPI:** `미배정 3 (기사 배정 필요)`, `지연 1 (일정 경과)`
- **기사 현황:** `김기사 [오늘 4 | 진행중 1 | 완료 2]`

### [T-02 작업함]
- **카드:** `[설치] | [기사 미배정] | OO마트 | 주소... | 14:00 | [기사 배정]`
- **모달:** `제목: 기사 배정`, `[기사를 선택해주세요 ▾]`, `[배정하기]`

### [T-03 상세 화면]
- **헤더:** `[설치] 기사 미배정`
- **배정:** `[김기사 (010-****-****) ▾] [재배정]`
- **상태:** `체크리스트: 완료 ✅`, `서명: 미완료 ⚠️`, `발송: 실패 ⚠️`

## 📂 연관 API 리스트
- `GET /api/teams/{teamId}/dashboard` (대시보드 통계)
- `GET /api/teams/{teamId}/workorders` (목록 필터링)
- `POST /api/workorders/{id}/assign-technician` (기사 배정/재배정)
- `GET /api/workorders/{id}/attachments` (증빙 데이터 조회)
