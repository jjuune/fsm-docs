    화면 지도(Screen Map)

    TL;DR

    • 역할별 화면과 문서 소스를 한 곳에서 매핑해, 개발 착수 범위를 빠르게 파악한다.

    화면맵 표기 규칙 (권장)

    1. 기본 단위는 “화면 코드” 기준으로 작성한다.
    2. 예: A-04, T-03, M-02
    3. 새로운 독립 화면만 새 화면 코드를 부여한다.
    4. 예: A-05 상품 분류 관리, A-06 상품 관리
    5. 기존 화면의 UI/섹션 확장은 새 코드로 만들지 않는다.
    6. 기존 화면 항목 아래 하위 bullet로 확장 메모를 추가한다.
    7. 확장 기능 표기는 아래 형식으로 통일한다.
    8. 확장(Gate 2 후보): [추가되는 기능/섹션]
    9. 예: 확장(Gate 2 후보): 선택 상품 조회 섹션(read-only) 추가
    10. 화면맵에서는 문서 생성 상태 표기(신규, Doc-xx)를 생략한다.
    11. 필요 시 별도 문서 인덱스/추적표에서 관리한다.
    12. 상세 스펙/텍스트 목업/API 문서는 화면맵 하위 문서에서 정리한다.
    13. 화면맵은 “전체 구조와 연결관계” 중심으로 유지한다.
